=== CRITICAL RULES (STRESS-TESTED AND VERIFIED) ===
1. PROJECT FIELD IN ACTIONS: The "project" field MUST be inside each action object in grok-proxy payloads, not just the query param. Without it, files land in the wrong directory and are unfindable.

2. ENCODING: When building grok-proxy URLs: JSON.stringify → btoa → encodeURIComponent → payload= param. The encodeURIComponent step is CRITICAL because base64 contains + and = characters that break URLs.

3. LARGE FILES (>2KB content): MUST use write_file_chunk. Split into ~1500-char chunks. Single write_file on large content means a URL >8KB which gets silently truncated and corrupts the file.

4. ENTRY POINT: Before writing App.tsx, ALWAYS read src/index.tsx first. Projects often import from 'components/App' not 'App', so the real file may be src/components/App.tsx. Writing to the wrong path has zero visible effect.

5. SEARCH_REPLACE ZERO MATCHES: If grok-write returns {zeroReplacements:true}, the exact string was NOT found. Read the file first, copy the exact text (including whitespace), then retry.

6. ALWAYS VERIFY: After any file write/delete, ALWAYS check get_console_errors to confirm no broken imports. A single delete_file that removes a file still imported elsewhere will crash the entire app.

7. DELETE SAFETY: Before deleting any file, read the importing file (usually App.tsx) to verify nothing depends on the file you're deleting. If something does, update the import first or write a replacement file first.

8. WINDOWS SHELL: run_command runs on Windows. Use PowerShell syntax for the limited commands that are allowed (npm, node, echo).

9. GIT LIMITATIONS: git reset --hard is sanitized to --soft by the bridge. To truly revert, you must manually restore files via write_file/delete_file.

10. BATCH LIMIT: Up to 10 actions per grok-proxy request. All execute sequentially.

11. ALL ENDPOINTS ARE GET: Do NOT attempt POST requests. Everything goes through GET URLs.

12. NEVER CLAIM SUCCESS WITHOUT VERIFICATION: Always confirm edits with grok-read or get_console_errors before telling the user it's done.
