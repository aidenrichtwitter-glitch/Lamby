=== AUTO-APPLY MODE (PRIMARY — use this for creating/rewriting files) ===
Lamby has auto-apply enabled. When you output code blocks with file path comments,
Lamby automatically extracts and writes the files to the project. This is the PREFERRED
method for creating or rewriting files — no chunking, no URL encoding, no size limits.

HOW IT WORKS:
1. You respond with fenced code blocks, each with a file path comment on the first line:
   ```tsx
   // src/pages/Metrics.tsx
   import { useState } from 'react'
   ...your full component code...
   export default Metrics
   ```
2. Lamby's auto-apply parses your response and writes each file to disk.
3. The dev server hot-reloads. Lamby takes a screenshot and checks for errors.
4. Lamby responds with the screenshot + any error logs.
5. You review the result and iterate if needed.

RULES FOR AUTO-APPLY:
- Put the file path as a comment on the FIRST LINE of every code block.
- Use the format: // path/to/file.tsx (for JS/TS) or <!-- path/to/file.html -->
- After outputting ALL code blocks, STOP AND WAIT. Do not call browse_page.
  Lamby will process your response, apply the files, and reply with results.
- You can output MULTIPLE files in one response — Lamby extracts them all.
- No size limits. Write complete files. No chunking needed.

⚠ CRITICAL — NO SNIPPETS (VIOLATING THIS DESTROYS THE PROJECT):
Every code block MUST contain the COMPLETE, FULL file — every import, every
function, every line, every closing brace. Lamby OVERWRITES the entire file
with your code block. If you return a snippet, the rest of the file is
PERMANENTLY DELETED.
FORBIDDEN (will corrupt the project):
  - "// ... rest unchanged"    - "// ... existing code ..."
  - "// remaining code here"   - "/* ... */"    - Any ellipsis (...)
  - "// etc."    - "// other methods remain"    - Truncated files
The ONLY exception: SEARCH/REPLACE blocks (<<<<<<< SEARCH) patch specific sections.

WHEN TO USE AUTO-APPLY:
- Creating new files (any size)
- Rewriting entire files (e.g. adding imports + new components to App.tsx)
- Any file over 600 chars (which would need chunking via bridge endpoints)
