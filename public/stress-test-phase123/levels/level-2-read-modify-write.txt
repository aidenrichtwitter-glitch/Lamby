=== OBJECTIVE: Read an existing file, make a small change, write it back, verify, then restore via git ===

You will modify ONE existing file using the read → modify → grok-create pattern.
This tests the most important skill: safely editing existing code.

IMPORTANT: Do NOT use grok-write (search/replace) for this test. Use grok-create to overwrite the entire file.

CRITICAL STOP RULES (READ THESE FIRST):
- You have a MAXIMUM of 15 browse_page calls for this entire test. If you exceed 15, STOP immediately.
- Once you post the FINAL coord note, you are DONE. Do NOT make any more browse_page calls.
- Do NOT retry steps after the restore. If restore succeeds, the test is DONE.
- Do NOT loop back to earlier steps after reaching step 7.
- If grok-create fails 2 times, use grok-write (search/replace) as a fallback. Do NOT retry grok-create more than 2 times.

STEPS (do them in order, post to coord after each):

1. DISCOVERY: Call /api/diag?project={{PROJECT}}
   → Post to coord EXACTLY: "L2-S1: diag OK, relay online"

2. READ THE TARGET FILE: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx

   → Post to coord EXACTLY: "L2-S2: read OK, [N] chars"
   IMPORTANT: Remember this exact char count — you will use it to verify the restore.

3. MODIFY THE CONTENT: Take the ENTIRE file content from step 2.
   Add this comment block at the very top of the file (before any imports):

   // This comment proves read-modify-write works correctly.

   Keep ALL existing code exactly as-is. Only add those 2 comment lines at the top.
   → Post to coord EXACTLY: "L2-S3: modified, comment added"

4. WRITE IT BACK: Use grok-create to overwrite src/pages/Dashboard.tsx with the modified content.
   CRITICAL: The content must include the ENTIRE original file plus your 2-line comment.
   Do NOT send just the comment — send the WHOLE file.
   URL: {{RELAY_BASE}}/api/grok-create?project={{PROJECT}}&path=src/pages/Dashboard.tsx&content=URL_ENCODED_FULL_CONTENT
   → Post to coord EXACTLY: "L2-S4: grok-create sent"
   → If grok-create fails: Try ONE more time. If it fails again, use grok-write as fallback.

5. VERIFY CONTENT: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx
   → Confirm the file starts with "// Bridge Test L2:"
   → Confirm the original code is still present
   → Post to coord EXACTLY: "L2-S5: verified, comment present"
   → If the comment is missing after 2 retries: Post "L2-S5: FAILED, moving to restore" and skip to step 7.
   DO NOT retry more than 2 times. Move on.

6. CONSOLE CHECK: Use grok-proxy (raw JSON) + get_console_errors
   → Post to coord EXACTLY: "L2-S6: console clean"

   Call: {{RELAY_BASE}}/api/grok-run?project={{PROJECT}}&cmd=git%20checkout%20HEAD%20--%20src/pages/Dashboard.tsx
   This restores the file to its exact state from the last git commit — guaranteed clean.
   Do NOT try to restore by re-writing the file content via grok-create. ONLY use git checkout.
   → Post to coord EXACTLY: "L2-S7: restored via git checkout"

   *** AFTER THIS STEP: NO MORE FILE WRITES. DO NOT CALL grok-create OR grok-write AGAIN. ***

8. VERIFY RESTORE: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx
   → Confirm the comment lines are GONE (file should NOT start with "//")
   → Confirm the char count matches the count from step 2
   → Post to coord EXACTLY: "L2-S8: restore verified, [N] chars"
   → If restore failed: Run step 7 ONE more time, then verify again. Do NOT loop.

9. FINAL: Post to coord EXACTLY: "L2 COMPLETE: PASS" or "L2 COMPLETE: FAIL - [reason]"

   *** TEST IS NOW DONE. STOP. DO NOT MAKE ANY MORE BROWSE_PAGE CALLS. ***
   *** DO NOT RETRY ANY STEPS. DO NOT WRITE ANY MORE FILES. YOU ARE FINISHED. ***
