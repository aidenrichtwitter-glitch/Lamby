=== LEVEL 2: READ-MODIFY-WRITE TEST ===
=== OBJECTIVE: Read an existing file, make a small change, write it back, verify, then restore via git ===

You will modify ONE existing file using the read → modify → grok-create pattern.
This tests the most important skill: safely editing existing code.

IMPORTANT: Do NOT use grok-write (search/replace) for this test. Use grok-create to overwrite the entire file.
This is the reliable approach that has been stress-tested and confirmed working.

STEPS (do them in order, post to coord after each):

1. DISCOVERY: Call /api/diag?project={{PROJECT}}
   → Post to coord: "L2-S1: diag OK, relay online"

2. READ THE TARGET FILE: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx
   → Study the content carefully. Note the EXACT char count and line count.
   → Post to coord: "L2-S2: Dashboard.tsx read OK, [N] chars, [N] lines"
   IMPORTANT: Remember this exact char count — you will use it to verify the restore later.

3. MODIFY THE CONTENT: In your response, take the ENTIRE file content from step 2.
   Add this comment block at the very top of the file (before any imports):
   // Bridge Test L2: Modified by Grok-4 at {{TIMESTAMP}}
   // This comment proves read-modify-write works correctly.

   Keep ALL existing code exactly as-is. Only add those 2 comment lines at the top.
   → Post to coord: "L2-S3: content modified, adding 2-line comment header"

4. WRITE IT BACK: Use grok-create to overwrite src/pages/Dashboard.tsx with the modified content.
   CRITICAL: The content must include the ENTIRE original file plus your 2-line comment.
   Do NOT send just the comment — send the WHOLE file.
   URL: {{RELAY_BASE}}/api/grok-create?project={{PROJECT}}&path=src/pages/Dashboard.tsx&content=URL_ENCODED_FULL_CONTENT
   → Post to coord: "L2-S4: grok-create sent for Dashboard.tsx, [N] chars"

5. VERIFY CONTENT: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx
   → Confirm the file starts with your "// Bridge Test L2:" comment
   → Confirm the rest of the file is unchanged from step 2
   → Post to coord: "L2-S5: Dashboard.tsx verified, comment present, original code intact"
   → If the comment is missing: Post failure, re-read, and retry steps 3-5

6. CONSOLE CHECK: Use grok-proxy (raw JSON) + get_console_errors
   → Post to coord: "L2-S6: console [clean/errors]"
   → If errors: The modification broke something. Read the error, fix the file, retry.

7. RESTORE ORIGINAL VIA GIT: Use grok-git to restore Dashboard.tsx to its committed state.
   This is MORE RELIABLE than trying to re-write the original content manually.
   Call: {{RELAY_BASE}}/api/grok-run?project={{PROJECT}}&cmd=git%20checkout%20HEAD%20--%20src/pages/Dashboard.tsx
   This restores the file to its exact state from the last git commit — guaranteed clean.
   Do NOT try to restore by re-writing the file content via grok-create. Use git.
   → Post to coord: "L2-S7: Dashboard.tsx restored via git checkout HEAD"

8. VERIFY RESTORE: Call /api/grok-read?project={{PROJECT}}&path=src/pages/Dashboard.tsx
   → Confirm the comment lines are GONE (file should NOT start with "//")
   → Confirm the char count matches the EXACT count from step 2
   → If char count doesn't match or comments are still present, the restore FAILED.
     In that case, re-run step 7 (git checkout) and re-verify.
   → Post to coord: "L2-S8: restore verified, [N] chars matches original [N] chars, no comment lines"

9. FINAL: Post to coord: "L2 COMPLETE: Read-modify-write cycle succeeded. File modified, verified, and restored via git checkout."

=== PASS CRITERIA ===
- All coord notes posted (L2-S1 through L2-S8 + final)
- Step 5 verification shows the comment was added correctly
- Step 6 console check is clean
- Step 8 verification shows the file was restored to its EXACT original state (matching char count from S2)
- No other files were modified
- No git commit was made (git checkout just reverts working tree changes)

=== FAIL CONDITIONS ===
- Dashboard.tsx content corrupted (missing code, truncated, garbled)
- Comment not present after write
- Original code not intact after modification
- File not properly restored in step 7 (char count mismatch)
- Comment lines still present after restore
- Console errors after modification
- Any other file modified
- Coord notes missing
