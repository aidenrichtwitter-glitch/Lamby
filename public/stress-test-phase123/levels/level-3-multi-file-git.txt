3a. Send ALL Metrics.tsx chunks ATOMICALLY (1 code execution — NOT browse_page):
    Write and execute a Python script that sends all chunks sequentially.
    This is ONE code execution action. No agents can interfere.

    ```python
    import urllib.request, urllib.parse, json, time

    RELAY = "{{RELAY_BASE}}"
    PROJECT = "{{PROJECT}}"
    PATH = "src/pages/Metrics.tsx"

    # Your full Metrics.tsx content from Plan A — paste it here as a Python string
    content = """YOUR METRICS.TSX CONTENT HERE"""

    # URL-encode the full content, then split into ~500-char chunks
    encoded = urllib.parse.quote(content, safe='')
    chunk_size = 500
    chunks = [encoded[i:i+chunk_size] for i in range(0, len(encoded), chunk_size)]
    total = len(chunks)

    print(f"Sending {total} chunks for {PATH}...")
    for i, chunk in enumerate(chunks):
        url = f"{RELAY}/api/grok-create-chunk?project={PROJECT}&path={PATH}&content={chunk}&chunk={i+1}&total={total}"
        try:
            req = urllib.request.Request(url)
            with urllib.request.urlopen(req, timeout=15) as resp:
                result = json.loads(resp.read().decode())
                status = result.get('results', [{}])[0].get('status', 'unknown')
                print(f"  Chunk {i+1}/{total}: {status}")
        except Exception as e:
            print(f"  Chunk {i+1}/{total}: ERROR - {e}")
        time.sleep(0.5)  # small gap between chunks

    print("All chunks sent!")
    ```

    *** The content variable must contain your COMPLETE Metrics.tsx code as a Python string.
