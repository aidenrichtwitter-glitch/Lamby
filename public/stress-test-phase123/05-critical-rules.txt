===============================================================
12 STRESS-TESTED RULES + KNOWN UNTESTED SCENARIOS
===============================================================

These rules were discovered through actual testing — each one caused
a real failure or silent corruption before being identified and fixed.

=== RULE 1: PROJECT FIELD IN ACTIONS ===
Status: BUG FIX (discovered during Phase 0)
The "project" field MUST be inside each action object in grok-proxy payloads.
  CORRECT: {"actions":[{"type":"write_file","project":"groks-app","path":"...","content":"..."}]}
  WRONG:   {"actions":[{"type":"write_file","path":"...","content":"..."}]} (project only in query param)
Without it: files go to wrong directory, unfindable by grok-read.

=== RULE 2: ENCODING (encodeURIComponent) ===
Status: BUG FIX (discovered during Phase 1, T1)
Base64 output contains + and = characters that break URL query params.
  MUST do: encodeURIComponent(btoa(json)) before putting in payload= param
  Without it: silent failures, payload corruption, random "action not found" errors

=== RULE 3: LARGE FILE SIZE LIMITS ===
Status: TESTED (Phase 3, size limit tests)
Relay accepts URLs up to at least 11KB (tested 10920 chars: PASS).
But Grok's browse_page has its own ~8KB limit.
  Safe rule for Grok: content < 5KB → write_file. Content > 5KB → write_file_chunk.
  Tested: 561B, 2.7KB, 3.3KB, 4KB, 5KB, 6KB, 8KB — all PASS at relay level.

=== RULE 4: ENTRY POINT GOTCHA ===
Status: DOCUMENTED (discovered during Phase 0)
src/index.tsx often imports from 'components/App' not 'App'.
Real App file: src/components/App.tsx (NOT src/App.tsx).
Writing to wrong path = zero visible effect. Always read index.tsx first.

=== RULE 5: SEARCH_REPLACE RESPONSE FORMAT ===
Status: TESTED (Phase 3, T4 + zero match test)
Success: {results:[{path:"file", replacements:1}]}
No match: {results:[{path:"file", replacements:0}]}
NOT {zeroReplacements:true} — that was incorrect in earlier docs.

=== RULE 6: ALWAYS VERIFY AFTER EDITS ===
Status: PROVEN BY INCIDENT (Phase 2 crash)
After any write/delete, ALWAYS call get_console_errors.
Returns {errors:[], totalErrors:0} when healthy.
If totalErrors > 0, something is broken — check imports.

=== RULE 7: DELETE SAFETY ===
Status: PROVEN BY INCIDENT (Phase 2 crash)
Deleting Analytics.tsx while App.tsx still imported it → ENTIRE APP CRASHED.
Vite error overlay: "Failed to resolve import '../pages/Analytics'"
FIX: Read importing file first. Update imports or write replacement BEFORE deleting.

=== RULE 8: WINDOWS SHELL WHITELIST ===
Status: TESTED (Phase 1 T6 + Phase 3 T16-19)
grok-run has a command whitelist:
  ALLOWED: npm (tested), node (tested), echo (tested)
  BLOCKED: dir, cd, type, Get-ChildItem, ls (all tested, all "Command not allowed")
  Param name: "command" (NOT "cmd")
Use grok-tree for file listing, grok-read for file reading.

=== RULE 9: GIT RESET LIMITATION ===
Status: TESTED (Phase 2, Step 1)
git reset --hard is sanitized to --soft by the bridge connector.
Cannot do destructive resets via bridge.
To truly revert: manually delete_file + write_file to restore files.

=== RULE 10: BATCH LIMIT ===
Status: DOCUMENTED (from relay source)
Up to 10 actions per grok-proxy request. All execute sequentially.

=== RULE 11: ALL ENDPOINTS ARE GET ===
Status: FUNDAMENTAL CONSTRAINT
POST requests are not possible through Grok's browse_page.
Every operation must go through GET URLs.

=== RULE 12: NEVER CLAIM SUCCESS WITHOUT VERIFICATION ===
Status: LESSON FROM PHASE 2
Always confirm edits with grok-read or get_console_errors.
Phase 2 showed that even a "successful" delete can break the app.

===============================================================
KNOWN UNTESTED SCENARIOS (exercise caution)
===============================================================
- Concurrent/parallel browse_page calls (bridge is serial queue — avoid)
- Bridge timeout/disconnect during long operations (30s relay timeout)
- Bulk operations: bulk_write, bulk_delete, apply_patch
- grok-interact actions: click, type, select (only evaluate tested)
- Low-usage proxy commands: move_file, copy_file, copy_folder, rename_file,
  archive_project, export_project, set_env_var, get_env_vars, rollback_last_change,
  and various analysis/generation commands
