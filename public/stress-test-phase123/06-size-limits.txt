=== GROK BROWSE_PAGE LIMIT (NOT DIRECTLY TESTED) ===

The relay itself has no practical URL length limit (tested up to 11KB).
However, Grok's browse_page tool has its own URL processing limit,
reported to be approximately ~8KB.

SAFE RULES FOR GROK:
  - Content < 5KB: Use write_file (single request). URL will be ~7KB, safely under limit.
  - Content > 5KB: Use write_file_chunk with ~1500-char chunks.
  - Always use encodeURIComponent on the base64 payload (adds ~10% to URL length).

=== URL LENGTH FORMULA ===

URL length ≈ relay_base_length + "?project=P&payload=" + encodeURIComponent(btoa(JSON.stringify({actions:[...]})))

For write_file:
  JSON overhead: ~80 chars (type, project, path fields)
  Base64 expansion: content × 1.33
  encodeURIComponent expansion: base64 × ~1.03 (only + and = get expanded)
  
  Approximate formula: URL ≈ 120 + (content_chars + 80) × 1.37

  Examples:
    1KB content → ~1600 char URL
    3KB content → ~4400 char URL
    5KB content → ~7000 char URL
    8KB content → ~11000 char URL
